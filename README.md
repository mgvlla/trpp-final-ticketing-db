# База данных для сервиса продажи билетов на мероприятия

Итоговый проект по дисциплине «Технологии разработки программных приложений».

## Описание проекта

Проект представляет собой реляционную базу данных, предназначенную для хранения и обработки информации, связанной с продажей билетов на мероприятия. База данных охватывает полный бизнес-процесс: от создания мероприятия организатором до фиксации оплаты покупателем.

В базе хранятся сведения о:
- категориях мероприятий (концерты, театры, лекции);
- организаторах и площадках проведения;
- пользователях сервиса;
- мероприятиях с датами и базовыми ценами;
- билетах с указанием мест, секторов, статусов;
- заказах, их составе и оплатах.

Схема базы данных нормализована до третьей нормальной формы (3НФ), что исключает избыточность данных и гарантирует их целостность.

## Технологический стек

- **СУБД:** PostgreSQL 16
- **Контейнеризация:** Docker Desktop
- **Язык:** SQL (DDL, DML, DQL)
- **Система контроля версий:** Git
- **Платформа разработки:** macOS / Linux

## Зависимости проекта

Для запуска проекта на локальной машине требуется наличие следующих программных компонентов:

- **Docker** версии 24.0 и выше
- **Docker Compose** версии 2.20 и выше

Дополнительные библиотеки не требуются. Образ PostgreSQL загружается автоматически из Docker Hub при первом запуске.

## Инструкция по запуску

### 1. Клонировать репозиторий

git clone https://github.com/mgvlla/trpp-final-ticketing-db.git
cd trpp-final-ticketing-db

### 2. Проверить наличие Docker

docker --version
docker compose version

### 3. Запустить контейнер с PostgreSQL

docker compose up -d

При первом запуске Docker загрузит образ postgres:16 и автоматически выполнит SQL-скрипты из папки sql/, создав структуру базы и заполнив её тестовыми данными.

### 4. Проверить работу контейнера

docker ps

Контейнер с именем trpp_final_postgres должен отображаться в статусе Up.

### 5. Подключиться к базе данных

docker exec -it trpp_final_postgres psql -U postgres -d event_ticketing_db

### 6. Просмотреть список таблиц

\dt

### 7. Выполнить демонстрационные запросы

docker exec -i trpp_final_postgres psql -U postgres -d event_ticketing_db < sql/03_queries.sql

### 8. Остановить контейнер

docker compose down

### 9. Полная переустановка (очистка данных)

docker compose down -v
docker compose up -d
