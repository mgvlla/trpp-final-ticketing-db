1. Dockerfile и упаковка приложения

Методические указания предписывают создать Dockerfile для упаковки приложения и обеспечить передачу внешних параметров через переменные окружения. Наш проект представляет собой базу данных на PostgreSQL. Для её развёртывания мы используем два взаимодополняющих инструмента: Dockerfile и docker-compose.yml.

Файл Dockerfile содержит инструкции для сборки образа приложения. Ключевые директивы:

— FROM postgres:16 — использование официального образа PostgreSQL 16 в качестве базового.

— ENV — все чувствительные параметры (имя базы данных, пользователь, пароль) передаются через переменные окружения. Это соответствует требованию методических указаний: приложение получает параметры для подключения к внешним ресурсам через переменные окружения.

— COPY ./sql /docker-entrypoint-initdb.d/ — копирование SQL-скриптов в специальную директорию контейнера. При первом запуске PostgreSQL автоматически выполняет все .sql-файлы из этой папки в алфавитном порядке.

— EXPOSE 5432 — объявление порта, на котором работает PostgreSQL.

Таким образом, Dockerfile полностью упаковывает приложение: при сборке образа SQL-скрипты становятся его неотъемлемой частью, а все параметры конфигурации вынесены в переменные окружения.

Для удобства локального запуска и демонстрации также используется docker-compose.yml, который дополняет Dockerfile. Docker Compose обеспечивает:

— Автоматическую загрузку образа postgres:16 из Docker Hub.

— Монтирование папки ./sql для инициализации базы данных.

— Сохранение данных в томе postgres_data.

— Проброс порта 5432 для подключения извне.

Оба файла используют переменные окружения для передачи параметров подключения, что полностью соответствует требованию методических указаний.

2. Развёртывание на облачной платформе

Данное требование помечено как необязательное и, согласно примечанию в методических указаниях, применимо только к проектам с бэкенд-частью. Наш проект является базой данных и не содержит веб-интерфейса или бэкенд-логики, поэтому данное требование к нам формально не применимо.

Тем не менее, разработанные Dockerfile и docker-compose.yml готовы к развёртыванию на любой облачной платформе, поддерживающей Docker.

3. Демонстрация развёртывания на локальной машине

Для проверки корректности развёртывания разработан файл TERMINAL_COMMANDS_MAC.md с пошаговыми командами. Полный цикл развёртывания включает: очистку предыдущего состояния, запуск контейнера с автоматической инициализацией базы данных, проверку статуса, подключение к базе данных, просмотр таблиц и содержимого, выполнение демонстрационных запросов, остановку и очистку.

4. Результат развёртывания

В результате выполнения docker compose up -d получаем работоспособную базу данных, доступную на порту 5432. Внутри неё:

— созданы все 12 таблиц;

— установлены все связи внешних ключей;

— применены все ограничения целостности (CHECK, UNIQUE, NOT NULL, ON DELETE CASCADE);

— база наполнена тестовыми данными (3 мероприятия, 2 организатора, 2 площадки, 3 пользователя, 7 билетов, 3 заказа, 2 оплаты);

— готовы к выполнению демонстрационные запросы из файла 03_queries.sql.

Развёртывание протестировано на компьютерах всех трёх участников команды. На всех машинах проект запустился корректно.

5. Вывод

В третьей части итогового проекта выполнено требование по упаковке приложения и его развёртыванию. Создан Dockerfile, запаковывающий SQL-скрипты в образ PostgreSQL. Все чувствительные параметры передаются через переменные окружения (ENV в Dockerfile и environment в docker-compose.yml).
